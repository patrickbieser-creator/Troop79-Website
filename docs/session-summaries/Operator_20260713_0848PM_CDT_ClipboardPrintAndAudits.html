Scout Clipboard: Print Layout Rebuilt From Scratch

Shipped: The printable Clipboard now fits exactly one sheet, double-sided (2 pages) — matching the troop's original Google-Sheets-based printout column-for-column, even for the most-decorated scout in the troop (Life rank, 32 merit badges, 7 ranks of history).

What started as "make it fit on 2 pages" surfaced a real, previously invisible bug: the printed Clipboard was actually 9 pages, because the site's nav/footer chrome bled into print, a mobile CSS breakpoint accidentally fired during printing and collapsed the rank grid, and CSS Grid doesn't paginate reliably in Chrome's print engine at all — tall content gets shoved onto blank fresh pages instead of flowing to fill space.

Root CauseFix
Site nav/footer printing in fullScoped @media print hide via :has(), targeted only at the Clipboard page
Mobile breakpoint firing during print (a Letter page is narrower than 900px)Scoped the breakpoint to screen only
CSS Grid pushing tall content onto blank pagesReplaced with a fixed, explicit print-only template — content is placed by hand into the exact column/page arrangement of the reference layout, not auto-flowed
Physical page-edge clipping headers on every pageAdded an explicit @page margin rule (asymmetric, extra top space)

Final layout: Page 1 — Scout+Tenderfoot / Second Class+Nights of Camping+Leadership / First Class+Service Projects. Page 2 — Star/Life/Eagle, then Merit Badges+Activities side by side. Verified via headless Chrome print-to-PDF against both a brand-new scout and the troop's most-progressed scout — both land at exactly 2 pages.

Decisions

[D-016] Star/Life/Eagle audit rules

Any logged leadership term counts as a qualifying Position of Responsibility (no rank-specific eligible-position list exists in the data); POR and time-in-rank checks are gated on the prior rank already being earned, merit-badge-count checks are not; only leadership terms dated on/after the prior rank's award count toward POR.

[D-017] Clipboard print layout: fixed template, not auto-flow

CSS Grid and CSS multi-column both proved unreliable for Chrome's print pagination. Built a separate print-only DOM structure with content manually placed to match the troop's original reference layout exactly, rather than trying to make one responsive structure serve both screen and print.
