What Was Accomplished

Leader Login Controls & Audit Trail Display

The Universal Ledger's entered_by stamp already existed but wasn't trustworthy — the login "Your name" field was free text, so anyone with the shared troop password could type any name. Added a datalist of authorized adults (auto-derived "First L." labels, disambiguated when two adults share one) and server-side validation rejecting unrecognized names. New can_login/login_name columns on leaders let an admin revoke a specific person's login or override their display name, without touching the shared password. Ledger list and detail views now show entered-by initials.

Roster: Click a Name for Full Detail

Scout and adult rows in the Roster report now open a detail dialog on click — everything the summary columns leave out (address, gender, swim class, YPT completion date, parent/guardian contacts) — with a deep link straight into the matching Lookups & Admin edit dialog instead of requiring a second manual search.

Scoutbook Export

Caught before it shipped wrong: Cross-referencing a real historical Scoutbook upload file against our own ledger data (matching by BSA Member ID + award date) proved the existing merit_badges.scoutbook_id values were wrong — populated in an earlier, unverified session. A user-provided "complete and verified" replacement CSV was also wrong for a 9-badge cluster (shifted by exactly +1), caught the same way and corrected after the user checked the source.

Built scoutbook_merit_badge_reference — a permanent, authoritative name→ID table covering 164 badge names (including ones the troop doesn't currently offer) — and corrected merit_badges.scoutbook_id plus added ranks.scoutbook_id from it. The new /admin/advancement/scoutbook-export page then does the actual work: a date-range preview of every merit badge/rank award ready to export, an explicit excluded-rows panel for anything unmappable (no BSA Member ID, no confirmed Scoutbook ID) instead of silently guessing, and a download verified byte-for-byte against the real sample file.

Duplicate Ledger Records Audit

New Audits check finds scouts credited twice for the same rank/merit badge award, requirement leaf, or other award (any date), or the same event/service/leadership entry on the same date — a naive grouping would have false-positived on recurring named events (an annual campout reuses its event code every year), so the grouping key is date-scoped only for kinds that legitimately repeat. Each duplicate group shows every conflicting record with full context, defaults to keeping the oldest, and resolves with the same soft-delete audit trail as a manual Universal Ledger delete.

First run surfaced 436 duplicate groups across 39 scouts in production — mostly genuine Fast Entry double-clicks, plus some overlap between the historical spreadsheet backfill and later live entries.

Decisions

[D-018] Leader login constrained via can_login/login_name on leaders

Reused the existing leaders table as the authorized-adult login allowlist instead of a second parallel table.

[D-019] Rename-cascade pattern for FK-adjacent identifiers

Renaming a requirement code or a leader's initials now cascades to every table that references it by convention or FK, verified against seeded test data before shipping.

[D-020] Tags moved into Lookups & Admin

One place for the troop's editable taxonomy instead of two.

[D-021] scoutbook_merit_badge_reference is the authoritative ID source

Built by cross-referencing real historical upload data against our own ledger, after two other claimed-authoritative sources both proved wrong for parts of the catalog.

[D-022] Scoutbook Export flags gaps instead of guessing

An explicit excluded-rows panel beats a silently incomplete or wrong export.

[D-023] Duplicate-detection grouping key varies by ledger kind

One-time achievements group by scout+kind+code alone; recurring event/service/leadership kinds also require the same date.
