Design Comps

News & Events CMS

Five clickable comps validating the visual direction and authoring UX for a hand-rolled news/events system — scouts draft articles through per-type templates, leaders review and publish. Built at wireframe fidelity using the site's real design tokens, not the final production code.

Placeholder content throughout is scouting-flavored but fictional. Each page carries its own judgment-call notes as HTML comments near the closing </body> tag.

Article Detail · News Type

News Template

Full /news/[slug] page for a News article: hero, headline, and a flexible body demonstrating a heading, paragraphs, a single captioned image, an uploaded photo gallery grid, a Gallery Link card (link-out to an external Google Photos album, shown side by side with the uploaded grid), a markdown table, and a responsive video embed.

Open template → Article Detail · Event Type

Event Template

Same flexible body, plus a date/time/location panel below the headline. Includes a live toggle to compare the with-Register-button state against the no-registration-link state.

Open template → Article Detail · Recognition Type

Recognition Template

Built around a Gallery Link centerpiece — a link-out card to the troop's external Google Photos album, since Court of Honor photo sets realistically live there, not in an in-house upload — plus a mocked-up (clearly-flagged optional/future) achievement list mapping scout name to rank or badge earned.

Open template → Authoring UX · Split-View

Event Editor (Write Here, See It There)

Two-pane authoring mockup: a form on the left (title, excerpt, type picker, date/time/location, a markdown body with insert-image/gallery/gallery link/video toolbar buttons and a syntax cheat-sheet) drives a genuinely live-rendered preview on the right, styled exactly like the Event template. Insert Gallery Link prompts for a Google Photos/Facebook URL and auto-detects the source label. Validates the "no rich-text editor — markdown plus structured fields plus live preview" approach.

Media Picker (Bunny CDN)

Upload New (drag-and-drop, with seeded success/error states) and Browse Existing (searchable, filterable thumbnail grid with single- or multi-select for galleries) — including a deliberate edge case where an existing photo is missing alt text.
